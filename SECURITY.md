# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes             |
| Older   | ❌ No              |

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public issue.
Instead, report it privately by emailing: **[your-email@example.com]**

Please include:
- A clear description of the vulnerability
- Steps to reproduce it
- Potential impact
- Your suggested fix (if any)

You can expect a response within **72 hours**.

## What Happens After You Report

1. We'll confirm receipt of your report
2. Investigate and assess the severity
3. Work on a fix and release a patch
4. Credit you in the release notes (if you'd like)

## Scope

This policy applies to the core calculator application code.
The following are **out of scope**:
- Issues in third-party libraries (report those to PyQt5 / Python directly)
- Feature requests or general bugs (use GitHub Issues instead)

## Dependencies

This project uses:
- **Python 3.x** — keep your Python installation up to date
- **PyQt5** — update regularly via `pip install --upgrade PyQt5`
